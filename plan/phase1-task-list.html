Schema Design Tasks (do these on paper / in a notes doc first)
Map your CSV columns to JSON fields for all 3 apps
DataOffline
▶

Open each CSV in VSCode and look at the headers. For each column, decide: what should this field be called in JSON? What type is it (string, number, array)? Are there columns you want to skip or rename? Write this mapping out before coding the converter.

  • Rename columns to camelCase: Total Calories → totalCalories
  • Decide on date format: use ISO 8601 "2024-03-15" consistently across all files
  • Numbers should be numbers in JSON, not strings: 1850 not "1850"
  • Empty CSV cells become null in JSON, not empty strings

CSV → JSON Conversion

Build one reusable converter tool in your scripts/ folder. Run it once per CSV file, output lands in data/. You'll never need to touch the CSV again.

Conversion Strategy
Audit your CSVs — note any cleaning needed before conversion
DataOffline
▶

Open each CSV in VSCode. Check for:

  • Date formats — are they consistent? Convert to YYYY-MM-DD
  • Number formats — commas as thousands separators (1,850 vs 1850)?
  • Missing values — empty cells, dashes, "N/A"?
  • Column headers — spaces, special characters, inconsistent casing?
  • Budget CSV specifically — is it rows=months, cols=categories? Or transposed?

Note every issue. The converter script will handle most of these, but knowing in advance means no surprises.

Convert budget CSV → budget.json and validate

▶

Start with the budget CSV — it has the most historical rows and will stress-test the converter. After converting:

  • Open the output JSON in VSCode — does it look right?
  • Count the rows: does JSON months[] count match CSV rows?
  • Check one known month against the CSV to verify values are correct
  • Validate the JSON: Ctrl+Shift+P → "Format Document" — VSCode will flag syntax errors
  • Move the validated file to data/budget.json
Repeat for calories CSV and recipes CSV
DataOffline
▶

Recipes may need manual cleanup after conversion — ingredient lists and methods are likely free text in your CSV, which needs care. Consider whether recipes are better entered by hand into the JSON if the CSV is poorly structured.

  • Calories: should be straightforward if CSV is rows = days
  • Recipes: check that ingredient data is parseable — may need manual formatting
  • Both: validate, check row counts, spot-check values

A — Replace Inline Data with fetch() from JSON

This is the pattern to add to each of your three existing apps. Replace whatever hardcoded data array you currently have with this loadData() function.

apps/03-budget-tracker/budget-tracker.html — <script> section
JavaScript 
// ── 1. CONFIG ────────────────────────────────────────────────
// Path from THIS html file to the data folder.
// Adjust if your folder structure differs.
const DATA_PATH = '../../data/budget.json';

// ── 2. LOAD DATA ─────────────────────────────────────────────
async function loadData() {
  try {
    const response = await fetch(DATA_PATH);

    // fetch() doesn't throw on 404 — check manually
    if (!response.ok) {
      throw new Error(`Could not load data: ${response.status} ${response.statusText}`);
    }

    const data = await response.json();
    return data;

  } catch (err) {
    // Show a visible error in the app rather than silent failure
    showError(err.message);
    return null;
  }
}

// ── 3. ERROR DISPLAY ─────────────────────────────────────────
function showError(message) {
  const el = document.getElementById('app-error');
  if (el) {
    el.textContent = `⚠ Data load failed: ${message}`;
    el.style.display = 'block';
  }
  console.error('Data load error:', message);
}

// ── 4. INIT — runs when page loads ───────────────────────────
async function init() {
  showLoading(true);

  const data = await loadData();
  if (!data) return; // error already shown

  // ── Pull out what you need ────────────────────────────────
  // These replace whatever was previously hardcoded
  const { meta, categories, budgets, months } = data;

  // ── Render ────────────────────────────────────────────────
  renderSummary(meta);
  renderCategories(categories);
  renderMonthlyChart(months, budgets);
  renderCurrentMonth(months, meta.currentMonth);

  showLoading(false);
}

function showLoading(visible) {
  const el = document.getElementById('loading');
  if (el) el.style.display = visible ? 'block' : 'none';
}

// ── 5. SAVE BACK TO JSON (Export button) ─────────────────────
// Since we can't write files directly from a browser,
// this creates a download of the updated JSON.
function exportData(data) {
  const json  = JSON.stringify(data, null, 2);
  const blob  = new Blob([json], { type: 'application/json' });
  const url   = URL.createObjectURL(blob);
  const a     = document.createElement('a');
  a.href      = url;
  a.download  = 'budget.json';
  a.click();
  URL.revokeObjectURL(url); // clean up
}

// ── 6. START ──────────────────────────────────────────────────
document.addEventListener('DOMContentLoaded', init);

⚠ Important: fetch() requires a server — it won't work if you double-click the HTML file directly (the file:// protocol blocks it). Always open your app via Live Server in VSCode (right-click the HTML → Open with Live Server). It serves on localhost, which fetch supports.

  <script>
  let convertedJSON = null;

  // Drag and drop
  const zone = document.getElementById('dropZone');
  zone.addEventListener('dragover', e => { e.preventDefault(); zone.classList.add('over'); });
  zone.addEventListener('dragleave', () => zone.classList.remove('over'));
  zone.addEventListener('drop', e => {
    e.preventDefault(); zone.classList.remove('over');
    readFile(e.dataTransfer.files[0]);
  });
  zone.addEventListener('click', () => document.getElementById('fileInput').click());
  document.getElementById('fileInput').addEventListener('change', e => readFile(e.target.files[0]));

  function readFile(file) {
    if (!file) return;
    const reader = new FileReader();
    reader.onload = e => convertCSV(e.target.result);
    reader.readAsText(file, 'UTF-8');
  }

  function convertCSV(csvText) {
    try {
      const delim    = document.getElementById('delimiter').value;
      const arrayKey = document.getElementById('arrayKey').value || 'rows';
      const lines    = csvText.trim().split('\n');
      const headers  = lines[0].split(delim).map(h => toKey(h));
      const rows     = [];

      for (let i = 1; i < lines.length; i++) {
        const cells = lines[i].split(delim);
        if (cells.every(c => !c.trim())) continue; // skip blank lines
        const obj = {};
        headers.forEach((key, idx) => {
          obj[key] = parseValue(cells[idx]);
        });
        rows.push(obj);
      }

      convertedJSON = {
        meta: {
          lastUpdated:   new Date().toISOString().split('T')[0],
          rowCount:      rows.length,
          generatedFrom: "csv-to-json converter"
        },
        [arrayKey]: rows
      };

      const pretty = JSON.stringify(convertedJSON, null, 2);
      document.getElementById('output').value = pretty;
      document.getElementById('status').textContent =
        `✓ Converted ${rows.length} rows, ${headers.length} columns`;
      document.getElementById('status').className = 'status';
      document.getElementById('downloadBtn').disabled = false;
    } catch (err) {
      document.getElementById('status').textContent = `Error: ${err.message}`;
      document.getElementById('status').className = 'status err';
    }
  }

  // Convert header string to camelCase key
  function toKey(str) {
    return str.trim()
      .replace(/[^a-zA-Z0-9 ]/g, '')
      .replace(/\s+(.)/g, (_, c) => c.toUpperCase())
      .replace(/^\w/, c => c.toLowerCase());
  }

  // Auto-detect and parse value types
  function parseValue(val) {
    if (!val || val.trim() === '' || val.trim() === '-' || val.trim() === 'N/A')
      return null;
    const clean = val.trim().replace(/,/g, ''); // remove thousands commas
    if (!isNaN(clean) && clean !== '') return Number(clean);
    return val.trim();
  }

  document.getElementById('downloadBtn').addEventListener('click', () => {
    if (!convertedJSON) return;
    const name = document.getElementById('filename').value || 'output.json';
    const blob = new Blob([JSON.stringify(convertedJSON, null, 2)],
                             { type: 'application/json' });
    const a = document.createElement('a');
    a.href = URL.createObjectURL(blob);
    a.download = name;
    a.click();
  });

Ongoing Commit Rhythm
Commit after each schema is finalised
GitOffline
▶

After you finish designing and documenting each JSON schema, make a commit. Small, focused commits are much more useful than one big commit at the end.

  • git add data/SCHEMA.md && git commit -m "Schema: budget.json structure defined"
  • git add data/ && git commit -m "Data: budget.json converted from CSV"
Commit monthly when you update budget.json with new data
Git
▶

After each monthly budget update, the commit message should include the month. This makes it trivially easy to recover any historical state.

  • git add data/budget.json && git commit -m "Budget update: May 2025 actuals"
  • Then push to GitHub: git push
Use branches for app feature work
Git
▶

When building a new feature, create a branch. This keeps main always in a working state. Merge back when the feature is complete and tested.

  • git checkout -b feature/budget-yoy-charts — create branch
  • ... do your work, commit as you go ...
  • git checkout main && git merge feature/budget-yoy-charts — merge when done
  • git branch -d feature/budget-yoy-charts — clean up
